# refactor-instructions.md テンプレート

このテンプレートに沿って、ルート直下の `refactor-instructions.md` を生成する。
`{...}` は分析結果で埋める。各セクションは必須。

人間がそのまま保存し、実装担当モデル（Codex / Opus など）に次の形で渡せる内容にする：

> 「/goal refactor-instructions.md に書かれたことを完遂しろ」

## Stop And Ask Conditions

以下に該当したら実装を止め、ユーザーに確認する：

- 認証 / 課金 / 通知 / 外部連携に影響しうる
- 互換性を壊しうる
- 複数の設計案がありプロダクト判断が必要

## 生成時の注意

- `{...}` を残さず、必ず実際の分析結果で埋める。
- Debt Map の各負債には必ず「根拠（ファイルパス）」を入れる。証拠のない負債を書かない。ツール検出（knip 等）は grep で裏取りしてから載せ、false positive は「削除禁止」として明記する。
- 「提案のみ」の負債は Implementation Phases の実装対象に入れない（Phase 7 / Out-of-scope に分類する）。
- Stop And Ask に該当する項目は「実装前に確認すべき質問」へ確実に回す。実装判断で足りるものは質問にしない。
- Baseline の実測値は分析時点で必ず取る。実測なしの Baseline Commands 表を作らない。
- Non-Negotiables にはプロジェクト固有の規約（CLAUDE.md / AGENTS.md / docs から発見したもの: マイグレーションツールの一本化、コミット規約、触ってはいけないディレクトリ等）を反映する。
